## Data Migration Details

### Migration Process

The `migrate-to-postgres.ts` script:

1. **Initializes PostgreSQL** - Creates all tables and indexes
2. **Connects to SQLite** - Reads from `.data/paper-trading.db`
3. **Migrates data** - Row by row with type conversion
4. **Verifies integrity** - Checks constraints and relationships
5. **Reports statistics** - Shows what was migrated

### What Gets Migrated

- ✅ Account data (balance, P&L)
- ✅ All positions (open and closed)
- ✅ All orders (pending and filled)
- ✅ Position history (for backtesting)
- ✅ Paper trading positions
- ✅ Instrument list (100 NIFTY stocks)
- ✅ All indexes

### Data Integrity

- No data loss during migration
- Type conversions handled automatically
- Relationships and constraints preserved
- Timestamps maintained
- Original SQLite database untouched

### Transaction Safety

```typescript
// Migration uses transactions
await client.query("BEGIN");
try {
  // Insert all data
  // ... 
  await client.query("COMMIT");
} catch (error) {
  await client.query("ROLLBACK");
  // Data remains unchanged
}
```

## Troubleshooting

### PostgreSQL Connection Issues

**Problem:** `Error: connect ECONNREFUSED 127.0.0.1:5432`

**Solutions:**
```bash
# Start PostgreSQL (macOS)
brew services start postgresql@15

# Start PostgreSQL (Linux)
sudo systemctl start postgresql

# Start PostgreSQL (Docker)
docker start trading-db

# Check if running
lsof -i :5432
```

### Migration Script Errors

**Problem:** `Migration failed: Connection error`

**Check:**
```bash
# Verify SQLite database exists
file .data/paper-trading.db

# Verify PostgreSQL is running
psql -h localhost -U postgres -c "SELECT 1"

# Check credentials
echo "POSTGRES_URL=$POSTGRES_URL"
```

### Authentication Issues

**Problem:** `FATAL: Ident authentication failed`

**Fix:**
```bash
# Reset password
psql -h localhost -U postgres -c \
  "ALTER USER admin WITH PASSWORD 'admin';"
```
