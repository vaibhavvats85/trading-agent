# PostgreSQL Migration - Quick Start

Complete migration from SQLite to PostgreSQL in 3 steps.

## Quick Setup (5 minutes)

### Step 1: Prepare PostgreSQL

**Option A: Using Homebrew (macOS)**
```bash
brew install postgresql@15
brew services start postgresql@15
```

**Option B: Using Docker (Any OS)**
```bash
docker run --name trading-db \
  -e POSTGRES_USER=admin \
  -e POSTGRES_PASSWORD=admin \
  -e POSTGRES_DB=trading_agent \
  -p 5432:5432 \
  -d postgres:15
```

### Step 2: Run Setup Script

```bash
cd /Users/vaibhavvats/Documents/Apps/Agents/trading-agent-nextjs

# Make script executable
chmod +x setup-postgres.sh

# Run setup with your credentials
./setup-postgres.sh localhost 5432 admin admin
```

This will:
- ✅ Create PostgreSQL database
- ✅ Create user account
- ✅ Set up permissions
- ✅ Generate .env.postgres.local configuration

### Step 3: Migrate Data

```bash
# Copy configuration to .env
cat .env.postgres.local >> .env

# Run migration
npm run migrate-to-postgres
```

## Verify Setup

```bash
# Check PostgreSQL connection
psql -h localhost -U admin -d trading_agent -c "SELECT COUNT(*) FROM instruments;"

# Should return: count = 100
```

## Start Application

```bash
# The application will use PostgreSQL automatically
npm run dev
```

## Troubleshooting

### PostgreSQL won't start

```bash
# macOS
brew services restart postgresql@15

# Linux
sudo systemctl restart postgresql

# Docker
docker start trading-db
```

### Connection refused on port 5432

```bash
# Check if PostgreSQL is running
lsof -i :5432

# If nothing shows, start it:
brew services start postgresql@15  # macOS
sudo systemctl start postgresql     # Linux
docker start trading-db             # Docker
```

### User authentication failed

```bash
# Reset password
psql -h localhost -U postgres -c \
  "ALTER USER admin WITH PASSWORD 'admin';"
```

### Migration errors

```bash
# Verify SQLite database exists
file .data/paper-trading.db

# Check PostgreSQL connection
psql -h localhost -U admin -d trading_agent -c "SELECT 1"

# Re-run migration
npm run migrate-to-postgres
```

- **`setup-postgres.sh`** - Automated setup script

## Environment Variables

```bash
# .env file (after migration)
DATABASE_TYPE=postgres
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DATABASE=trading_agent
POSTGRES_USER=admin
POSTGRES_PASSWORD=admin
```

## Rollback to SQLite

```bash
# Update .env
echo "DATABASE_TYPE=sqlite" >> .env

# Restart application
npm run dev

# All SQLite data is preserved in .data/paper-trading.db
```

## For Deployed Environments

If deploying to cloud (Railway, Heroku, Supabase):

```bash
# Use connection string in .env
DATABASE_TYPE=postgres
POSTGRES_URL=postgresql://user:password@host.example.com/database
```
